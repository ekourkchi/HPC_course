SimGrid installation procedure

The Download section of the SimGrid Site should have all the information you need to install SimGrid.

Download the .tar.gz of the latest stable version from the above Website.

Once you have downloaded simgrid, a typical installation process on, say, your own Ubuntu box, would go as:

  • cd simgrid
  • sudo apt-get install boost
    • In case the Boost library is not installed on your system
  • sudo apt-get install g++
    • In case you don’t have a C++ compiler installed
  • cmake -DCMAKE_INSTALL_PREFIX=/usr/local -Denable_smpi=on -Denable_documentation=off
    • To generate all Makefiles
  • make
    • To compile SimGrid
  • make check
    • To run some tests
  • ctest
    • To run more tests
  • sudo make install
    • To copy SimGrid files to /usr/local/bin, /usr/local/lib, etc.

If you are not superuser on the system (e.g., if you want to run simulations on our Cray cluster), then you have to install SimGrid in your home directory, with the modified cmake invocation:

  • cmake -DCMAKE_INSTALL_PREFIX=$HOME/MyInstalls/ -Denable_smpi=on -Denable_documentation=off
    • To generate all Makefiles
